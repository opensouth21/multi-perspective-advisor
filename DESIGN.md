# 多视角思维顾问 · 项目设计说明书

> **技能名称**：Multi-Perspective Advisor（多视角思维顾问）  
> **所属项目**：Nuwa Skill 生态系统  
> **创建日期**：2026-06-17  
> **当前版本**：v1.0.0  
> **关联文件**：`SKILL.md`（运行时行为规范）  

---

## 一、设计背景与动机

### 1.1 问题

单个 Perspective Skill（如乔布斯、马斯克、芒格）虽然在其专长领域提供了深度洞察，但面对**复杂、多维度、跨领域**的实际问题时，单一视角存在天然盲区：

- 商业决策者可能低估尾部风险
- 技术创新者可能忽视市场博弈
- 投资分析师可能对用户行为理解不足

### 1.2 解决方案

创建一个**统一的调度层 Skill**——"多视角思维顾问"，其核心价值不在于提供新观点，而在于：

1. **智能选择**：根据问题类型，从 14 个独立 Perspective 中挑选最合适的 4-8 个
2. **跨域互补**：强制包含至少 1 个不同领域的视角，确保盲区覆盖
3. **结构化合并**：按主题归并共识与分歧，而非简单罗列
4. **可操作输出**：将各视角语言翻译成统一、具体的行动建议

### 1.3 定位

该 Skill 是一个**聚合层（Aggregation Layer）**，不是替代者。它的存在是为了让 14 个独立 Perspective 技能产生"1+1>2"的效果。

## 二、架构设计

### 2.1 视角分类体系

14 个 Perspective 技能分为 5 大类别：

| 类别 | 数量 | 视角 | 核心心智模型 |
|------|------|------|-------------|
| 科学/认知 | 4 | 费曼、Karpathy、Ilya、芒格 | 教学、技术可信度、AI 安全、跨学科思维 |
| 商业/战略 | 5 | 马斯克、乔布斯、张一鸣、PG、Naval | 成本拆解、产品设计、系统效率、创业、财富杠杆 |
| 创意/内容 | 2 | MrBeast、孙宇晨 | 注意力经济、投机套利 |
| 风险/哲学 | 2 | 塔勒布、特朗普 | 尾部风险、权力博弈 |
| 实战/方法论 | 1 | 张学峰 | 教育规划、职业选择 |

**设计原则**：分类不是固定的。当新问题类型出现时，可动态增删视角并重新归类。

### 2.2 选择逻辑架构

```
问题输入 → 问题分类 → 选择同领域深度视角(1-2个)
                              → 选择不同领域互补视角(≥1个，强制)
                              → 去重/防重叠检查
                              → 输出 4-8 个视角列表
```

**选择准则**：
- **深度优先**：至少 1 个与问题同领域的核心专家视角
- **多样性保证**：至少 1 个不同领域的视角（强制要求）
- **防重叠**：避免选择视角高度相似的组合
- **数量控制**：4-8 个，最少 4 个保证多样性，最多 8 个防止信息过载

### 2.3 分析合并流程

```
独立视角 A 分析 ──┐
独立视角 B 分析 ──┼──→ 共识提取 → 综合洞察
独立视角 C 分析 ──┤
独立视角 D 分析 ──┘
独立视角 E 分析 ──→ 分歧标注 → 冲突分析
独立视角 F 分析 ──→ 盲区识别 → 遗漏警告
独立视角 G 分析 ──→ 行动翻译 → 具体建议
```

**输出结构**：
1. 核心结论（共识摘要）
2. 维度一：主题名称 → 共识/分歧
3. 维度二：主题名称 → 共识/分歧
4. 盲区提醒
5. 行动建议

## 三、技术实现

### 3.1 文件结构

```
multi-perspective-advisor/
├── SKILL.md          # 运行时行为规范（用户指令 + 执行协议）
├── DESIGN.md         # 项目设计说明书（本文档）← 你在看的这个
└── CHANGELOG.md      # 版本变更记录
```

### 3.2 视角激活机制

支持两种方式：

**方式 1：Skill 工具加载（推荐）**
```
Skill(tool: "elon-musk-perspective")
Skill(tool: "charlie-munger-perspective")
```

**方式 2：Agent 内部模拟（兜底）**
当对应 Perspective 无法通过 Skill 工具加载时，模拟其核心心智模型、决策启发式和语言风格进行分析。

### 3.3 依赖关系

- 依赖 14 个 Perspective Skill 存在于 `~/.workbuddy/skills/nuwa-skill/examples/`
- 如果某个 Perspective 不存在，Skill 会自动降级为该视角的简化分析

## 四、质量控制指标

### 4.1 有效性指标

| 指标 | 标准 | 检测方法 |
|------|------|----------|
| 视角数量 | 4-8 个 | 检查清单 |
| 跨域覆盖率 | ≥1 个不同领域 | 分类校验 |
| 共识/分歧标注 | 每个维度都有 | 输出结构检查 |
| 行动建议可操作性 | 非抽象陈述 | 语义分析 |

### 4.2 性能指标

- 单次响应时间：1500-4000 字
- 视角激活成功率：≥80%（支持模拟降级）
- 错误恢复：当超过 2 个视角无法加载时，通知用户

## 七、注意事项

1. **DESIGN.md 与 SKILL.md 的区别**：
   - `DESIGN.md` 记录**设计决策、架构、演变历史**（适合回顾和迭代）
   - `SKILL.md` 记录**运行时行为规范**（每次调用时自动加载执行）
   - 更新技能时应同步更新 DESIGN.md

2. **视角独立性**：每个 Perspective Skill 独立维护，本 Skill 不直接修改它们，只做选择和整合。

3. **用户确认**：重大架构变更（如新增/移除视角类别）应先与用户讨论。
